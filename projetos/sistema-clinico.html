Sistema para Clinicas

Este projeto consiste em um Sistema de Gestão para Clínica Médica, desenvolvido em Python, com foco em organização de dados, lógica de negócio e experiência do usuário em ambiente de linha de comando (CLI).O sistema simula necessidades reais de uma clínica, permitindo o gerenciamento de pacientes, médicos e consultas, com persistência local de dados e análise estatística básic

Contexto

O processo de agendamento de serviços era realizado de forma manual, o que gerava desorganização, conflitos de horários, perda de informações e retrabalho no atendimento ao cliente.

Objetivo

Desenvolver um sistema funcional e bem estruturado capaz de: Cadastrar pacientes e médicos Armazenar dados localmente com segurança Aplicar regras de negócio para consultas normais e emergenciais Gerar estatísticas relevantes dos pacientes Oferecer uma experiência clara e organizada no terminal

Solução Desenvolvida

O projeto foi estruturado de forma modular, priorizando legibilidade, manutenção e separação de responsabilidades. Persistência de Dados Os dados são armazenados em arquivos JSON, permitindo simplicidade e fácil leitura: pacientes.json medicos.json O sistema trata cenários como arquivos inexistentes ou vazios, garantindo robustez.

Tecnologias Utilizadas

  • Python
  • JSON
  • Lógica de Programação
  • Programação modular
  • Tratamento de execuções

Resultado e Aprendizados

Sistema funcional e estável Código organizado e legível Fácil manutenção e expansão Experiência de uso acima da média para aplicações CLI Este projeto consolidou conceitos essenciais como: Estruturação de sistemas Validação de dados Lógica de negócio Persistência de informações Experiência do usuário além do front-end web
